# 密码输入框焦点问题 - 快速测试指南

## 🎯 核心测试（必做）

### 测试 1: 学生修改密码
```
1. 登录学生账号
2. 点击右上角 "Change Password"
3. 在 "Current Password" 中连续输入 10 个字符
4. 在 "New Password" 中连续输入 10 个字符
5. 在 "Confirm Password" 中连续输入 10 个字符

✅ 预期: 可以连续输入，无需重复点击
```

### 测试 2: 管理员重置密码
```
1. 登录管理员账号
2. 进入 Admin > Students
3. 点击任意学生的紫色钥匙图标
4. 在 "New Password" 中连续输入 10 个字符
5. 在 "Confirm Password" 中连续输入 10 个字符

✅ 预期: 可以连续输入，无需重复点击
```

## 🔍 详细测试（推荐）

### 测试 3: 密码显示/隐藏
```
1. 打开密码修改模态框
2. 输入 5 个字符
3. 点击眼睛图标（显示密码）
4. 继续输入 5 个字符
5. 再次点击眼睛图标（隐藏密码）
6. 继续输入 5 个字符

✅ 预期: 焦点始终保持，可以连续输入
```

### 测试 4: 密码强度指示器
```
1. 打开密码修改模态框
2. 在 "New Password" 中逐字符输入: p-a-s-s-w-o-r-d-1-2-3
3. 观察密码强度条变化

✅ 预期: 
- 输入流畅，无焦点丢失
- 强度条实时更新
- 颜色从红色变为绿色
```

### 测试 5: 错误验证
```
1. 打开密码修改模态框
2. 在 "New Password" 中输入: pass (少于8字符)
3. 点击提交
4. 修改为: password (没有数字)
5. 点击提交
6. 修改为: password123 (有效)

✅ 预期: 
- 每次输入都流畅
- 错误提示正确显示
- 焦点保持在输入框
```

## 🐛 问题排查

### 如果焦点仍然丢失
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 重启开发服务器
   ```bash
   cd frontend
   npm run dev
   ```
3. 硬刷新页面（Ctrl+Shift+R 或 Cmd+Shift+R）

### 如果看到控制台错误
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签
3. 截图错误信息
4. 检查是否有 React 警告

### 如果功能异常
1. 检查文件是否正确保存
2. 确认没有语法错误
3. 重新启动开发服务器
4. 清除 .next 缓存
   ```bash
   cd frontend
   rm -rf .next
   npm run dev
   ```
